## ARQUITECTURA DE DATOS (MODELO RELACIONAL)
- Entidad Usuario: Long id, String nombre, String email, String password, Boolean activo. Relación: tiene muchos perfiles, autor de muchos tópicos y respuestas.
- Entidad Perfil: Long id, PerfilNombre nombre (Enumeración: ROLE_ADMIN, ROLE_USER).
- Entidad Curso: Long id, String nombre, String categoria, Boolean activo.
- Entidad Topico: Long id, String titulo, String mensaje, LocalDateTime fechaCreacion, StatusTopico statusTopico, Boolean activo (Borrado lógico / Soft Delete vía @SQLRestriction). Relaciones: Pertenece a un Curso, creado por un Usuario (autor), contiene lista de Respuestas.
- Entidad Respuesta: Long id, String mensaje, LocalDateTime fechaCreacion, Boolean solucion, Boolean activo. Relaciones: Vinculado a un Tópico, creado por un Usuario (autor).
- Enumeración StatusTopico: NO_RESPONDIDO, NO_SOLUCIONADO, SOLUCIONADO, CERRADO.

## FLUJO OPERATIVO Y SEGURIDAD (RBAC)
- Autenticación: Stateless mediante Spring Security y tokens JWT cifrados usando PasswordEncoder (BCrypt).
- Endpoint Login: POST /login (Público, devuelve el JWT Token).
- Roles y Permisos:
  * ROLE_ADMIN: Control total del foro, administración de usuarios, cursos, perfiles y supervisión de logs de IA.
  * ROLE_USER: Participación en la comunidad (Crear tópicos, responder, marcar solución).
- Swagger UI de Documentación: https://foro-hub-christian.duckdns.org/api/swagger-ui/index.html (Permite autorizar mediante Bearer Token).

## MODERACIÓN INTELIGENTE MULTICAPA (FAILOVER DE IA)
Para mitigar fallos de red o límites de cuotas, el Foro Hub procesa los textos mediante tres niveles en cascada:
1. Capa Primaria (Cloud): Google Gemini (Modelo: Gemini-3-Flash-Preview) para análisis semántico profundo y detección de insultos o spam.
2. Capa Secundaria (Cloud - Failover): Mistral AI (Modelo: Mistral Large) que se activa automáticamente si Gemini reporta latencia o caída del servicio.
3. Capa de Seguridad Local (Offline): Filtro Heurístico basado en código Java puro que aplica reglas de negocio, análisis de palabras clave y puntuación de riesgo si ambos proveedores de IA están caídos.
Observabilidad: Implementation del patrón Decorator (LoggingChatModelDecorator) para capturar latencias, tokens de entrada y salida, y metadata exacta de cada llamada a las APIs de IA.
